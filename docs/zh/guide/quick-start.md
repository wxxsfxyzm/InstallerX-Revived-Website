# 快速开始

欢迎使用 InstallerX Revived！本应用设计为“开箱即用”，无需繁琐的初始化设置。但理解 **主页**、**配置** 与 **设置** 的核心概念，将帮助您发挥它的最大潜能。

## 1. 初次运行与权限

首次安装后，您无需立即进行复杂的设置。当您第一次尝试安装应用时，InstallerX 会按需索要以下必要权限：

* **通知权限：** 用于显示安装进度和结果以及前台服务通知。
* **所有文件访问权限：** 用于读取/删除存储空间中的 APK/APKS/XAPK/APKM 文件。
* **授权器权限 (Shizuku/Root)：** 用于执行静默安装或特权操作。

## 2. 核心概念：主页 / 配置 / 设置

InstallerX Revived 将主要功能分为了三个标签页：

* **主页:** 展示默认安装器状态、授权/特权状态、配置数量和设备信息；默认安装器相关页面也从主页状态卡进入。
* **配置:** 定义安装或卸载请求如何处理（例如：安装模式、安装标记、指定用户、授权覆盖等）。
* **设置:** 定义全局的应用偏好（例如：界面主题、对话框与通知行为、黑名单、实验室等）。

### 授权方式
默认情况下，应用使用 **Shizuku** 进行授权。主页的授权页面会检测设备上所有可供 InstallerX 使用的授权方式，并允许您选择一种作为全局授权，用于整个应用中需要提权的操作，例如锁定安装器、安装应用、刷写模块、删除文件或执行系统级设置。这里仅检测可用性；真正需要权限时，应用才会发起对应的授权请求。应用显示的功能也会根据当前选择变化。

可选授权方式包括：
* **Root:** 能够以最高权限执行 InstallerX 的所有特权操作，但由于需要冷启动 `app_process`，性能通常略弱于 Shizuku。
* **Shizuku:** 根据 Shizuku 的激活方式获得 shell 或 root 对应权限。推荐大多数用户使用，通常比直接使用 Root 响应更快。
* **Dhizuku:** 基于 DevicePolicyManager API 的设备所有者模式。可完成锁定安装器、安装应用等部分操作，但面对其他需要更高特权的场景能力有限。
* **无:** 完全受系统限制。InstallerX 作为系统安装器运行时，可以获得静默安装应用的能力；作为普通用户应用运行时，会走系统安装会话，并由系统包管理器弹出确认。

::: tip 配置覆盖
主页中选择的授权方式用于决定 InstallerX 自身执行提权操作时使用什么方式。“配置”中的授权器则用于安装应用，默认跟随主页中为应用设定的授权方式，也可以在某个单独的“配置”中强制指定为不同的授权方式（例如全局用 Shizuku，但某个特定配置用 Root）。
:::

## 3. 分应用配置 (配置与作用域)

InstallerX 最强大的功能之一是能够根据 **安装来源应用**（即发起安装请求的那个 App）来应用不同的安装规则。

* **默认配置 (Default):** 列表中的第一个配置。它是全局默认生效的，任何没有被特殊指定的应用都会使用此配置。
* **新建配置:** 点击右下角的 **+ 新增** 按钮即可创建自定义配置。
* **设定作用域:** 如果您希望对不同来源的应用区别对待（例如：让 *酷安* 自动安装，但让 *文件管理器* 弹出确认框）：
    1.  新建或编辑一个配置。
    2.  点击 **作用域**。
    3.  勾选您希望该配置生效的一个或多个应用。
    4.  保存后，当这些应用发起安装请求时，就会自动应用该配置。

## 4. 安装模式

您可以在“配置”中选择四种安装展现形式：

* **对话框:** 弹出包含应用详情的对话框，需手动点击“安装”。
* **对话框自动:** 弹出对话框，但自动进行安装。
* **通知栏:** 在后台安装，通过标准通知栏进度条显示进度，需要手动确认。
* **通知栏自动:** 同上，但无需用户手动确认，进一步减少交互干扰。

### Android 16 实时活动 (Live Activity)
对于运行 **Android 16+** 的设备，InstallerX Revived 支持全新的 **实时活动 (Live Activity)** 通知 API，可在锁屏和状态栏提供更优雅的实时进度展示。

**支持的系统包括：**
* ColorOS 16+
* OneUI 8.0+
* AOSP 16 QPR1+
* Pixel 16 QPR1+
* HyperOS 3.0.300+

## 5. 关于锁定安装器

由于国内定制系统对安装器的严格管控和系统魔改，应用内自带的“锁定为默认安装器”功能可能存在兼容性问题。

* **兼容性警告：** 该功能可能在 **部分 HyperOS**，以及 **所有的 ColorOS 和 OriginOS** 上无效。
* **解决方案：** 在上述系统中如果无法锁定安装器，请搭配基于 LSPosed 的 **[InxLocker](https://github.com/Chimioo/InxLocker)** 模块进行锁定，以获得最佳体验。

## 6. 常见技巧与特定设备说明

::: info 小米 / HyperOS 用户注意
**1. 伪装安装者：**
在 HyperOS 上更新或安装系统应用时，系统要求“安装者”必须也是系统应用。默认情况下，InstallerX Revived 会自动将 **安装者包名** 伪装为 `com.android.shell` 以确保兼容性。除非您是高级用户，否则无需修改。

**2. 自动锁定：**
如果您发现安装应用后，安装器锁定状态丢失（变回系统默认），请在首页点击默认安装器状态卡，并在 **默认安装器** 页面开启 **“自动锁定安装器”** 开关。
:::

::: tip 海外 / 非 CN 地区用户
如果您在非中国大陆地区使用，且感觉安装速度较慢（通常是 Play Protect 在扫描），可以尝试开启设置中的 **“停用系统安装验证”** 开关。
:::
