# 实验室

实验室只保留当前仍属于实验性质或高风险的功能。这些选项可能随版本调整、移除，或在不同 ROM 上表现不同。建议只开启自己确实需要的功能。

## 模块刷写

开启 **启用模块刷写** 后，InstallerX 可以从 ZIP 文件安装支持的 Magisk、KernelSU 或 APatch 模块。

当 ZIP 同时包含应用安装包和模块信息时，InstallerX 会让你选择作为应用安装还是作为模块安装。模块安装是 InstallerX 少数可能使用模块专用命令执行的场景，不完全等同于普通 PackageInstaller 安装。

::: danger
刷写前请确认模块来源可靠。错误模块可能导致系统组件异常或无法正常开机。
:::

## Root 实现方式与字符画

启用模块刷写时，需要选择与当前设备环境匹配的 Root 实现方式：

* **Magisk**
* **KernelSU**
* **APatch**

你也可以开启安装开始时显示 InstallerX 字符画。系统安装器模式下是否始终使用 Root 执行特权操作，已经移动到 **设置 -> 安装器设置 -> 授权器参数微调**。

## HTTP 安全策略

联网版支持通过分享下载直链安装 APK，也支持在线更新。实验室中的 HTTP 安全策略控制是否允许明文 HTTP：

* **仅允许安全连接 (HTTPS)：** 最安全，推荐默认使用。
* **允许本地明文 (Localhost)：** 仅允许本地或局域网明文 HTTP。
* **允许所有：** 风险最高，只适合完全信任来源和网络时使用。

::: warning
允许所有明文 HTTP 会降低安全性。只有在你完全信任下载来源和当前网络时才建议使用。
:::

联网相关设置只会在联网版中显示。离线版不申请联网权限，使用联网功能时会给出明确错误。

## GitHub 更新通道

联网版可以选择 GitHub 更新通道：

* **官方：** 直接访问 GitHub。
* **7ED 代理：** 使用内置代理通道。
* **自定义 GHProxy：** 填写自己的代理 URL。

自定义代理 URL 为空时，会回到官方通道。

## 使用建议

日常使用建议保持默认设置。需要排查或测试时，一次只开启一个实验功能，并先用安全 APK 或可信模块验证；如果 ROM 开始跳回系统安装器、安装会话提前失败，或联网下载被阻止，就关闭相关选项。
